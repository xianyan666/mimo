# 408考研学习助手（AI Workflow版）

## 项目背景

本项目旨在构建一个基于命令行（CLI）的 408 考研学习助手。

用户每天完成学习后，通过命令行向系统汇报学习内容。

系统负责：

* 记录学习进度
* 总结学习成果
* 保存长期学习记忆
* 分析学习趋势
* 提供后续学习建议

本项目不追求复杂的教学功能，而是聚焦于 AI Workflow 的设计与实现。

---

# 项目目标

实现一个能够长期跟踪用户 408 学习过程的 CLI Agent。

用户通过命令：

```bash
python cli.py submit
python cli.py today
python cli.py check
```

完成学习记录提交与查询。

系统通过本地文件维护长期记忆。

---

# 学习范围

仅统计计算机考研 408 内容：

* 数据结构
* 计算机组成原理
* 操作系统
* 计算机网络

非 408 内容默认不纳入统计。

---

# 核心功能

## 1. submit

提交今日学习记录。

记录内容：

* 学习科目
* 学习章节
* 学习时长
* 完成内容
* 遇到的问题
* 自我评价

---

## 2. today

查看今日学习记录。

输出：

* 今日学习内容
* 学习时长
* 完成情况

---

## 3. check

检查整体学习进度。

输出：

* 各科累计学习时长
* 学习次数
* 最近学习内容
* 薄弱知识点

---

## 4. review（扩展）

生成阶段性复盘报告。

输出：

* 高频错误
* 重复遗忘知识点
* 学习趋势分析

---

# 长期记忆设计

长期记忆采用本地 JSON 文件。

memory.json：

```json
{
  "records":[]
}
```

每条记录包含：

* 日期
* 科目
* 章节
* 学习时长
* 学习内容
* 问题
* AI总结

---

# 边界条件

## 输入为空

提示重新输入。

---

## 非408内容

提示：

该内容不属于408范围，是否保存为备注。

---

## 学习时长异常

学习时长：

* 小于0
* 大于16小时

要求用户确认。

---

## 重复提交

同一天允许多次提交。

采用追加模式。

---

## 无历史记录

check命令提示：

暂无学习记录。

---

## 记忆文件损坏

自动备份：

memory_backup.json

重新创建记忆文件。

---

## LLM不可用

自动降级为模板总结模式。
