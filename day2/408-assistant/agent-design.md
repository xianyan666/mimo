# 学生学习训练辅助 Agent 设计

## 1. 设计目标

本 Agent 用于辅助学生完成学习训练任务，主要功能包括：

- **资料检索**：从课程资料、学习笔记、作业文档中检索与问题相关的内容片段
- **知识点解释**：基于检索到的资料，对知识点进行解释说明
- **练习生成**：根据指定知识点、难度和数量生成练习题
- **答案批改**：对学生提交的答案进行批改，给出判断和理由
- **错题分析**：根据错题记录和学习记录分析学生的薄弱知识点
- **学习建议**：根据学习记录和薄弱点生成复习建议和学习计划
- **结果校验**：对最终回答进行校验，确保基于资料、不越界、无明显幻觉

核心原则：回答必须基于已有资料，资料不足时必须明确说明无法确定，不凭空编造内容。

---

## 2. 使用场景

### 场景 1：查询课程知识点

学生提问"什么是 LRU 页面置换算法？"，Agent 从课程资料中检索相关内容，基于资料给出解释。如果资料中没有相关内容，必须提示"资料中未找到相关信息，无法确定"。

### 场景 2：根据知识点生成练习题

学生要求"帮我出 5 道关于页面置换算法的中等难度题"，Agent 基于课程资料生成练习题，每道题必须包含题目、答案和解析。

### 场景 3：批改学生答案

学生提交答案后，Agent 对比参考答案进行批改，给出分数和判断理由。不能只给分数，必须说明扣分原因。

### 场景 4：分析错题和薄弱知识点

根据学生的错题记录和学习记录，分析出薄弱知识点，并给出针对性的复习建议。不根据单个错误过度推断，数据不足时不能强行得出结论。

### 场景 5：生成学习计划

根据薄弱知识点、学习目标和可用时间，生成具体的学习计划。计划必须与学习目标相关，不能超出可用时间。

### 场景 6：资料不足时提示无法确定

当学生问的问题在课程资料中找不到相关内容，或者检索结果相似度低于阈值时，Agent 必须明确提示"资料不足，无法确定"，而不是编造答案。

## 3. 总体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                           User                                  │
│                     （学生输入问题/指令）                          │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                          Router                                 │
│                   （判断任务类型，分发请求）                        │
│                                                                 │
│   任务类型：knowledge_query / generate_exercise / check_answer  │
│            / weakness_analysis / learning_plan / file_check     │
└──────┬──────────────┬──────────────┬────────────────────────────┘
       │              │              │
       ▼              ▼              ▼
┌────────────┐ ┌────────────┐ ┌────────────┐
│  RAG 检索  │ │  工具调用   │ │  文件检查   │
│            │ │            │ │            │
│ rag_search │ │ generate   │ │ file       │
│            │ │ _exercise  │ │ _status    │
│            │ │ check      │ │ _check     │
│            │ │ _answer    │ │            │
│            │ │ weakness   │ │            │
│            │ │ _analyzer  │ │            │
│            │ │ learning   │ │            │
│            │ │ _plan      │ │            │
│            │ │ _generator │ │            │
└──────┬─────┘ └──────┬─────┘ └──────┬─────┘
       │              │              │
       ▼              ▼              ▼
┌─────────────────────────────────────────────────────────────────┐
│                       Validator                                 │
│                   （结果校验，判断回答是否有效）                     │
│                                                                 │
│   校验内容：是否有资料依据 / 是否越界 / 是否存在幻觉              │
│            / 是否需要补充说明                                     │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                        Answer                                   │
│                   （输出最终回答给用户）                           │
└─────────────────────────────────────────────────────────────────┘
```

**模块说明：**

- **User**：学生用户，通过自然语言输入问题或指令
- **Router**：路由模块，分析用户输入的任务类型，将请求分发给对应的工具
- **RAG 检索**：从课程资料、学习笔记、作业文档中检索相关资料片段
- **工具调用**：调用具体的工具完成任务（生成练习、批改答案、分析薄弱点、生成计划）
- **文件检查**：检查指定的课程资料文件是否存在、是否可读
- **Validator**：校验模块，对工具返回的结果进行校验，确保回答基于资料、不越界、无幻觉
- **Answer**：将校验通过的回答输出给用户

## 4. 调用流程

### Step 1：函数流设计

```
用户输入
    │
    ▼
Router 判断任务类型
    │
    ├── 知识点查询 ──→ rag_search ──→ result_validator ──→ 输出
    │
    ├── 生成练习题 ──→ rag_search + generate_exercise ──→ result_validator ──→ 输出
    │
    ├── 批改答案 ──→ rag_search + check_answer ──→ result_validator ──→ 输出
    │
    ├── 错题分析 ──→ weakness_analyzer ──→ result_validator ──→ 输出
    │
    ├── 学习计划 ──→ learning_plan_generator ──→ result_validator ──→ 输出
    │
    └── 文件检查 ──→ file_status_check ──→ 输出
```

**流程说明：**

1. 用户输入问题或指令
2. Router 分析输入内容，判断属于哪种任务类型
3. 根据任务类型调用相应的 RAG 检索或工具
4. 对工具返回的结果调用 result_validator 进行校验
5. 校验通过后输出最终回答；校验不通过时，补充说明或提示不确定

## 5. 工具定义

### 5.1 rag_search

**功能说明：**

从课程资料、作业文档、学习记录、错题记录中检索与用户问题相关的资料片段。使用向量相似度检索，返回最相关的 top_k 个片段。

**输入参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| query | string | 是 | 用户查询的自然语言文本 |
| top_k | int | 否 | 返回结果数量，默认 5 |

**输入示例：**

```json
{
  "query": "什么是 LRU 页面置换算法？",
  "top_k": 5
}
```

**输出结果：**

| 字段 | 类型 | 说明 |
|------|------|------|
| chunks | array | 检索到的资料片段列表 |
| chunks[].content | string | 片段内容 |
| chunks[].source | string | 来源文件名 |
| chunks[].score | float | 相似度分数，范围 0-1 |

**输出示例：**

```json
{
  "chunks": [
    {
      "content": "LRU 是最近最久未使用页面置换算法，每次淘汰最长时间未被访问的页面。",
      "source": "操作系统学习笔记.md",
      "score": 0.86
    },
    {
      "content": "页面置换算法包括 FIFO、LRU、LFU 等，其中 LRU 需要记录每个页面的最后访问时间。",
      "source": "操作系统作业.md",
      "score": 0.72
    }
  ]
}
```

**边界条件：**

- 只从已有课程资料中检索，不凭空补充资料中不存在的事实
- 当最高相似度分数低于阈值（如 0.3）时，必须提示资料不足
- 不处理与学习训练无关的问题（如"今天天气怎么样"）
- 检索结果按相似度分数降序排列
- 不修改原始资料文件

**失败处理：**

| 失败场景 | 处理方式 |
|----------|----------|
| 没有检索结果 | 返回空 chunks 列表，提示"未检索到相关资料" |
| 检索结果相似度过低（最高分 < 0.3） | 返回结果但标记 low_confidence=true，提示"资料匹配度较低，以下内容仅供参考" |
| 多个资料来源内容冲突 | 返回所有冲突片段，标记 conflicts=true，提示"资料中存在不同说法，请注意甄别" |
| 资料文件无法读取 | 跳过该文件，记录错误日志，在输出中标注哪些文件未能检索 |
| 索引未建立 | 提示"资料索引尚未建立，请先构建索引" |

### 5.2 file_status_check

**功能说明：**

检查指定的课程资料、作业文件、错题文件、学习记录文件是否存在，是否可以读取，是否为空文件。

**输入参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file_name | string | 是 | 要检查的文件名 |

**输入示例：**

```json
{
  "file_name": "操作系统学习笔记.md"
}
```

**输出结果：**

| 字段 | 类型 | 说明 |
|------|------|------|
| exists | bool | 文件是否存在 |
| readable | bool | 文件是否可读 |
| empty | bool | 文件是否为空 |
| message | string | 状态描述信息 |

**输出示例：**

```json
{
  "exists": true,
  "readable": true,
  "empty": false,
  "message": "文件可正常读取"
}
```

**边界条件：**

- 只检查学习资料相关文件，不读取无关的私人文件
- 不修改文件内容，只做只读检查
- 文件路径必须在允许的资料目录范围内

**失败处理：**

| 失败场景 | 处理方式 |
|----------|----------|
| 文件不存在 | exists=false, readable=false，提示"文件不存在" |
| 文件无权限读取 | exists=true, readable=false，提示"文件存在但无读取权限" |
| 文件为空 | exists=true, readable=true, empty=true，提示"文件为空，无内容" |
| 文件格式不支持（如二进制文件） | exists=true, readable=true，提示"文件格式不支持，仅支持文本格式" |
| 文件名包含路径遍历字符（如 ../） | 拒绝操作，提示"文件名不合法" |

### 5.3 generate_exercise

**功能说明：**

根据指定知识点、难度和数量生成练习题。每道题必须包含题目、答案和解析。

**输入参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| topic | string | 是 | 知识点名称 |
| difficulty | string | 否 | 难度等级：easy/medium/hard，默认 medium |
| count | int | 否 | 生成题目数量，默认 5 |

**输入示例：**

```json
{
  "topic": "LRU 页面置换算法",
  "difficulty": "medium",
  "count": 5
}
```

**输出结果：**

| 字段 | 类型 | 说明 |
|------|------|------|
| exercises | array | 练习题列表 |
| exercises[].question | string | 题目内容 |
| exercises[].answer | string | 参考答案 |
| exercises[].analysis | string | 解析说明 |

**输出示例：**

```json
{
  "exercises": [
    {
      "question": "给定页面访问序列 7,0,1,2,0,3,0,4，物理块数为 3，使用 LRU 算法，请计算缺页次数。",
      "answer": "缺页次数为 6 次。",
      "analysis": "初始状态为空，依次装入 7,0,1（缺页3次）；访问 2 时淘汰 7（缺页1次）；访问 0 命中；访问 3 时淘汰 1（缺页1次）；访问 0 命中；访问 4 时淘汰 2（缺页1次）。共缺页 6 次。"
    }
  ]
}
```

**边界条件：**

- 必须基于已有课程资料或明确的知识点生成，不凭空编造知识点
- 题目数量不能超过设定上限（如 20 道）
- 难度只能为 easy、medium、hard，不接受其他值
- 不生成与学习无关的内容
- 生成的题目必须包含答案和解析，不能只有题目

**失败处理：**

| 失败场景 | 处理方式 |
|----------|----------|
| 知识点资料不足 | 提示"该知识点资料不足，无法生成高质量练习题"，返回可用的部分题目 |
| 难度参数非法 | 提示"难度参数必须为 easy、medium 或 hard" |
| count 超出上限 | 提示"题目数量超出上限（最大 20 道），已调整为 20 道" |
| 无法生成有效答案 | 跳过该题，不输出没有答案的题目 |
| 知识点不在课程范围内 | 提示"该知识点不在课程范围内" |

### 5.4 check_answer

**功能说明：**

批改学生提交的答案，对比参考答案给出判断和理由。

**输入参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| question | string | 是 | 题目内容 |
| student_answer | string | 是 | 学生提交的答案 |
| reference_answer | string | 否 | 参考答案（可从资料检索） |

**输入示例：**

```json
{
  "question": "什么是 FIFO 页面置换算法？",
  "student_answer": "FIFO 是优先淘汰最早进入内存的页面。",
  "reference_answer": "FIFO 页面置换算法会淘汰最早进入内存的页面。"
}
```

**输出结果：**

| 字段 | 类型 | 说明 |
|------|------|------|
| correct | bool | 答案是否正确 |
| score | int | 分数，0-100 |
| feedback | string | 批改反馈说明 |

**输出示例：**

```json
{
  "correct": true,
  "score": 95,
  "feedback": "答案基本正确，表达清楚。'优先淘汰'的表述可以更准确为'淘汰最早进入内存的页面'。"
}
```

**边界条件：**

- 必须有题目和学生答案，否则无法批改
- 尽量基于参考答案或检索资料批改，不凭主观判断
- 不能只给分数，必须说明原因
- 对于开放性题目，需说明判断依据

**失败处理：**

| 失败场景 | 处理方式 |
|----------|----------|
| 学生答案为空 | 提示"学生答案为空，无法批改" |
| 缺少参考答案 | 尝试从资料检索参考答案；检索不到则提示"缺少参考答案，无法准确批改" |
| 题目表述不清 | 提示"题目表述不清，无法判断答案是否正确" |
| 无法判断正误 | correct=null, score=null，提示"无法确定答案是否正确"，并说明原因 |

### 5.5 weakness_analyzer

**功能说明：**

根据错题记录和学习记录分析学生的薄弱知识点，输出薄弱点列表和改进建议。

**输入参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| wrong_questions | array | 是 | 错题记录列表 |
| wrong_questions[].topic | string | 是 | 知识点名称 |
| wrong_questions[].error_type | string | 是 | 错误类型（如"计算错误"、"概念混淆"） |
| learning_records | array | 否 | 学习记录列表 |
| learning_records[].topic | string | 是 | 知识点名称 |
| learning_records[].duration | int | 是 | 学习时长（分钟） |
| learning_records[].result | string | 否 | 学习结果描述 |

**输入示例：**

```json
{
  "wrong_questions": [
    {
      "topic": "页面置换算法",
      "error_type": "计算错误"
    }
  ],
  "learning_records": [
    {
      "topic": "操作系统",
      "duration": 60,
      "result": "LRU 计算不熟练"
    }
  ]
}
```

**输出结果：**

| 字段 | 类型 | 说明 |
|------|------|------|
| weak_points | array | 薄弱知识点列表 |
| suggestions | array | 改进建议列表 |

**输出示例：**

```json
{
  "weak_points": [
    "LRU 页面置换算法计算",
    "缺页次数统计"
  ],
  "suggestions": [
    "重新复习页面置换算法的执行过程",
    "完成 3 道同类计算题"
  ]
}
```

**边界条件：**

- 只分析学习相关数据，不处理无关信息
- 不根据单个错误过度推断，至少需要多次错误才能判定为薄弱
- 数据不足时不能强行得出结论，应提示数据不足
- 分析结果应基于统计规律，而非主观臆断

**失败处理：**

| 失败场景 | 处理方式 |
|----------|----------|
| 错题记录为空 | 提示"无错题记录，无法分析薄弱知识点" |
| 学习记录为空 | 仅基于错题分析，标注"无学习记录参考" |
| 数据格式不正确 | 提示"数据格式错误，请检查输入" |
| 分析结果置信度低 | 提示"数据量不足，分析结果仅供参考" |

### 5.6 learning_plan_generator

**功能说明：**

根据薄弱知识点、学习目标和剩余时间生成具体的学习计划。

**输入参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| weak_points | array | 是 | 薄弱知识点列表 |
| goal | string | 是 | 学习目标 |
| available_days | int | 是 | 可用天数 |

**输入示例：**

```json
{
  "weak_points": ["LRU 页面置换算法", "银行家算法"],
  "goal": "复习操作系统考试",
  "available_days": 7
}
```

**输出结果：**

| 字段 | 类型 | 说明 |
|------|------|------|
| plan | array | 学习计划列表 |
| plan[].day | int | 第几天 |
| plan[].task | string | 当天任务 |

**输出示例：**

```json
{
  "plan": [
    {
      "day": 1,
      "task": "复习页面置换算法基本概念，完成 3 道 LRU 计算题"
    },
    {
      "day": 2,
      "task": "复习银行家算法，理解安全序列的判断方法"
    },
    {
      "day": 3,
      "task": "完成页面置换算法综合练习，对比 FIFO/LRU/LFU 区别"
    }
  ]
}
```

**边界条件：**

- 学习计划必须与学习目标相关
- 计划不能超出可用时间范围
- 任务量不能明显不合理（如一天安排 20 小时学习）
- 每天的任务应具体可执行

**失败处理：**

| 失败场景 | 处理方式 |
|----------|----------|
| 缺少学习目标 | 提示"请指定学习目标" |
| 剩余时间无效（如 0 或负数） | 提示"可用天数必须大于 0" |
| 薄弱点为空 | 提示"无薄弱知识点，无法生成针对性计划" |
| 无法生成合理计划 | 提示"无法在给定时间内生成合理计划，请调整参数" |

### 5.7 result_validator

**功能说明：**

对最终回答进行校验，判断是否基于资料、是否越界、是否存在不确定内容、是否需要补充说明。

**输入参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| answer | string | 是 | 待校验的回答内容 |
| sources | array | 是 | 回答依据的资料来源列表 |
| task_type | string | 是 | 任务类型 |

**输入示例：**

```json
{
  "answer": "LRU 是最近最久未使用算法，每次淘汰最长时间未被访问的页面。",
  "sources": ["操作系统学习笔记.md"],
  "task_type": "knowledge_query"
}
```

**输出结果：**

| 字段 | 类型 | 说明 |
|------|------|------|
| valid | bool | 回答是否有效 |
| issues | array | 发现的问题列表 |
| final_answer | string | 校验后的最终回答 |

**输出示例：**

```json
{
  "valid": true,
  "issues": [],
  "final_answer": "LRU 是最近最久未使用算法，每次淘汰最长时间未被访问的页面。"
}
```

**边界条件：**

- 必须检查回答是否有资料依据
- 必须检查是否回答了与学习无关的问题
- 必须检查是否存在明显幻觉（编造不存在的内容）
- 不确定时必须要求说明"不确定"或"资料不足"
- 检查回答是否与问题匹配

**失败处理：**

| 失败场景 | 处理方式 |
|----------|----------|
| 缺少来源 | valid=false，提示"回答缺少资料依据" |
| 回答与问题不匹配 | valid=false，提示"回答内容与问题不匹配" |
| 回答超出资料范围 | 在回答中标注"以下内容超出了资料范围，仅供参考" |
| 检测到内容冲突 | 标注冲突内容，提示"资料中存在不同说法" |

## 6. 结果校验规则

### 规则 1：回答必须基于检索资料

所有知识点查询类回答，必须有对应的资料来源。如果回答中的关键信息无法在资料中找到依据，必须标注为"推测"或"不确定"。

### 规则 2：资料不足时必须明确说明

当 rag_search 返回空结果或相似度低于阈值时，不能编造答案。必须明确说明"当前资料中未找到相关信息，无法确定"。

### 规则 3：工具调用失败时不能伪造结果

任何工具调用失败（如文件不存在、参数错误、超时），必须如实返回错误信息，不能用伪造的数据替代。

### 规则 4：生成练习题必须包含答案和解析

generate_exercise 返回的每道题必须包含 question、answer 和 analysis 三个字段。如果无法生成有效答案，该题不应出现在输出中。

### 规则 5：批改答案必须给出原因

check_answer 返回结果必须包含 feedback 字段，说明批改理由。不能只返回 correct=true/false 而不解释原因。

### 规则 6：学习计划必须符合用户给出的时间范围

learning_plan_generator 生成的计划天数不能超过 available_days 参数。每天的任务量不能超过合理范围（如不超过 12 小时）。

### 规则 7：对冲突资料必须提示存在冲突

当 rag_search 检索到多个来源的资料内容存在矛盾时，必须在输出中标注冲突，并列出不同来源的说法，让学生自行判断。

### 规则 8：与学习训练无关的问题应拒绝

当用户提问与学习训练无关（如"帮我写一篇小说"、"今天吃什么"），应提示"该问题不在学习辅助范围内"，不作答。

### 规则 9：不能过度推断

weakness_analyzer 不能仅凭一次错误就判定某个知识点为薄弱。需要多次错误或结合学习记录综合判断。

### 规则 10：校验结果必须记录校验过程

result_validator 的输出必须包含 issues 列表，记录校验过程中发现的所有问题，即使最终 valid=true，也应记录已检查的项目。

## 8. 示例完整调用链

**用户输入：**

> "我总是算错 LRU 页面置换算法，帮我分析并出几道题。"

**调用流程：**

**Step 1：Router 判断任务类型**

Router 分析用户输入，识别出两个子任务：
- 薄弱点分析（"我总是算错" → 需要分析错题记录）
- 练习生成（"出几道题" → 需要生成练习题）

**Step 2：rag_search 检索 LRU 相关资料**

```
rag_search(query="LRU 页面置换算法")
```

返回：
```json
{
  "chunks": [
    {
      "content": "LRU 是最近最久未使用页面置换算法，每次淘汰最长时间未被访问的页面。",
      "source": "操作系统学习笔记.md",
      "score": 0.92
    }
  ]
}
```

**Step 3：weakness_analyzer 分析错题记录**

```
weakness_analyzer(
  wrong_questions=[{topic:"LRU 页面置换算法", error_type:"计算错误"}],
  learning_records=[{topic:"操作系统", duration:60, result:"LRU 计算不熟练"}]
)
```

返回：
```json
{
  "weak_points": ["LRU 页面置换算法计算", "缺页次数统计"],
  "suggestions": ["重新复习页面置换算法的执行过程", "完成 3 道同类计算题"]
}
```

**Step 4：generate_exercise 生成练习题**

```
generate_exercise(topic="LRU 页面置换算法", difficulty="medium", count=3)
```

返回 3 道练习题，每题包含 question、answer、analysis。

**Step 5：result_validator 校验回答**

```
result_validator(
  answer="根据分析，你在 LRU 页面置换算法的计算上存在薄弱...",
  sources=["操作系统学习笔记.md"],
  task_type="weakness_analysis_and_exercise"
)
```

校验通过，valid=true。

**Step 6：输出最终答案**

向用户输出：
1. 薄弱点分析结果（基于 weakness_analyzer 的输出）
2. 3 道 LRU 练习题（基于 generate_exercise 的输出）
3. 每道题附带答案和解析

## 9. 总结

本 Agent 设计将资料检索、工具调用、结果校验三个环节结合起来，构建了一个完整的学生学习训练辅助系统。

**核心价值：**

- **资料检索能力**：通过 rag_search 工具，从课程资料中检索相关内容，确保回答有据可依
- **工具调用能力**：通过 7 个工具覆盖学习训练的主要场景（查询、出题、批改、分析、计划）
- **结果校验能力**：通过 result_validator 和 10 条校验规则，确保回答的准确性和可靠性
- **边界和失败处理**：每个工具都定义了明确的边界条件和失败处理方式，不仅考虑 happy path，也覆盖了异常场景

**可靠性保障：**

- 资料不足时明确提示"无法确定"，不编造内容
- 工具调用失败时如实返回错误，不伪造结果
- 对冲突资料进行标注，让学生自行判断
- 与学习无关的问题直接拒绝，不越界回答
- 分析结果标注置信度，避免过度推断

该 Agent 可以有效辅助学生进行学习训练，帮助学生查询知识点、练习巩固、发现薄弱环节并制定改进计划。
