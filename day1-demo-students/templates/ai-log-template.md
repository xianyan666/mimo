# AI Log — [项目名称]

> 每条记录包含五个字段。最关键的是「人工判断」——必须写出你采纳/修改/拒绝了 AI 的什么建议以及为什么。
> 如果人工判断只写"AI说得对"或为空，该条记录在评审中无效。

---

## 第 N 条：[简短标题，如"Spec 定义中的 AI 协作"]

| 字段 | 内容 |
|------|------|
| **目的** | [这次问 AI 想达成什么？] |
| **输入** | [给 AI 提供了什么材料/约束？] |
| **建议** | [AI 给出了什么方案？] |
| **人工判断** | [你采纳/修改/拒绝了 AI 的什么建议？为什么？] |
| **验证** | [如何证明最终结果可靠？] |

---

## 示例（仅供参考）

| 字段 | 内容 |
|------|------|
| **目的** | 请 AI 审查我的 Spec 是否有遗漏 |
| **输入** | 提供了 course-faq.md 和当前 spec.md 草稿（目标4条、非目标3条、验收3条） |
| **建议** | AI 建议增加"用户登录功能"和"数据库存储历史记录" |
| **人工判断** | 拒绝了数据库和登录。——课程要求 CLI 工具、单用户本地运行，Spec 的非目标已明确写"不做数据库"和"不做用户登录"。AI 的建议违反了非目标约束 |
| **验证** | 重新检查 Spec 非目标列表——已有"不做数据库"和"不做用户登录"两条，AI 的建议确实是过度设计 |
