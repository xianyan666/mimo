# 项目一：Apache HTTP 服务器日志智能清洗与故障分析

> **难度等级**：Lv.2 入门级 | **总课时**：3 课时（每课时 45 分钟）
> **技术栈**：Python 3.10+ · pandas · re · matplotlib
> **数据文件**：`Apache.tar.gz`（解压后得到 Apache 日志文件，2000 条样本）

---

## 项目目标

你是运维工程师，Web 服务器报错，需要分析 Apache 错误日志找出故障原因。

借助 AI 完成全流程：**日志解析 → 异常识别 → 统计分析 → 可视化 → 报告输出**。

核心能力：正则表达式 · 数据解析 · 故障根因分析。

---

## 数据集

- **来源**：GitHub [logpai/loghub](https://github.com/logpai/loghub/tree/master/Apache)
- **背景**：Apache HTTP Server 运行时的错误日志
- **课堂数据**：`Apache.tar.gz`，解压后约 2000 条日志
- **日志格式**：`[时间戳] [日志级别] 日志内容`

```
[Sun Dec 04 04:47:44 2005] [notice] workerEnv.init() ok /etc/httpd/conf/workers2.properties
[Sun Dec 04 04:47:44 2005] [error] mod_jk child workerEnv in error state 6
```

---

## Lesson 1：日志探索与解析清洗

### 你要做什么

1. **解压数据**：解压 `Apache.tar.gz`，读取日志文件
2. **数据探索**：用 AI 辅助生成代码，统计总行数、各级别日志数量、时间范围、检查异常行
3. **正则解析**：编写正则表达式，从每条日志中提取 **3 个字段**：
   - `timestamp` — 时间戳
   - `level` — 日志级别（notice / error 等）
   - `content` — 日志内容
4. **结构化输出**：将解析结果保存为 `structured_logs.csv`
5. **理解大文件处理**：了解分块读取的思路

### 提交文件

| 文件 | 说明 |
|------|------|
| `lesson1_日志探索与解析.py` | 数据探索 + 解析流程脚本 |
| `log_parser.py` | 可复用的日志解析模块 |
| `structured_logs.csv` | 解析后的结构化数据 |

---

## Lesson 2：异常识别与统计分析

### 你要做什么

1. **理解错误类型**：学习 Apache 常见错误类型（error state 6、mod_jk、jk2_init 等）
2. **三维度筛选异常**：
   - 按日志级别筛选（error）
   - 按错误码/关键字筛选
   - 按模块筛选
3. **多维度统计分析**：
   - 时间维度：每日 Error 数量趋势
   - 类型维度：各错误码出现次数与占比
   - 模块维度：各模块异常率对比
4. **导出异常子集**：将筛选出的异常日志单独保存

### 提交文件

| 文件 | 说明 |
|------|------|
| `lesson2_异常识别与统计.py` | 异常识别 + 统计脚本 |
| `error_filter.py` | 异常筛选模块 |
| `statistics.py` | 统计分析模块 |
| `error_code_reference.csv` | 错误类型含义对照表 |

## Lesson 3：可视化展示与报告总结

### 你要做什么

1. **生成 3 种图表**（matplotlib）：
   - 时序图 — 每日 Error 数量变化趋势
   - 饼图 — 各错误类型占比分布
   - 柱状图 — 各模块 Error 数量对比
2. **封装工具库**：将前三节课的模块整合为 `log_analyzer/` 包
3. **撰写分析报告**：`analysis_report.md`

### 提交文件

| 文件 | 说明 |
|------|------|
| `lesson3_可视化与报告.py` | 可视化 + 工具库演示脚本 |
| `log_analyzer/` | 工具库目录（含 `__init__.py`、`log_parser.py`、`error_filter.py`、`statistics.py`、`visualizer.py`、`utils.py`、`README.md`） |
| `charts/` | 可视化图表 PNG 文件 |
| `analysis_report.md` | 分析报告（含故障根因分析 + 优化建议） |

### 分析报告要求

报告需包含以下章节，且必须有**你自己的分析思考**（不能全是 AI 写的）：

1. 项目概述
2. 数据探索
3. 日志解析方法
4. 异常识别结果
5. 统计分析结果
6. 可视化分析
7. 故障根因分析
8. 总结与系统优化建议

|------|------|
